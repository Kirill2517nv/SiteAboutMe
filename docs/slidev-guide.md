# Инструкция: добавление Slidev-презентаций к урокам

## Обзор

Slidev-презентации — интерактивные слайды на основе Markdown, которые собираются в статический HTML и раздаются Nginx. Django хранит ссылку на презентацию и управляет файлами-вложениями.

**Структура файлов урока:**
```
media/lessons/{Название_урока}/
├── задание.odt              # вложение (LessonAttachment)
├── пример-текста.odt        # вложение
├── presentation/             # Slidev dist (статический HTML)
│   ├── index.html
│   └── assets/
└── content/                  # изображения из блоков урока
```

## Шаг 1: Создать урок и загрузить файлы

1. Открыть Django Admin → Уроки → **Добавить урок**
2. Заполнить название (например, «LibreOffice Writer»)
3. В секции **Вложения** загрузить файлы (задание.odt, пример-текста.odt и т.д.)
4. Нажать **Сохранить**
5. **Запомнить ID урока и ID вложений** — они видны в URL админки и в списке вложений

## Шаг 2: Написать презентацию

Создать файл `slides.md` в корне проекта (или в отдельной папке).

**Минимальный шаблон:**
```markdown
---
theme: default
title: Название презентации
transition: slide-left
fonts:
  sans: Roboto
  mono: Fira Code
htmlAttrs:
  lang: ru
drawings:
  enabled: false
selectable: true
contextMenu: false
presenter: false
---

# Заголовок

Содержимое первого слайда

---

# Второй слайд

- Пункт 1
- Пункт 2
```

### Ссылки на файлы-вложения

Используй Django URL скачивания:

```markdown
📄 <a href="/lessons/{ID_урока}/file/{ID_вложения}/">задание.odt</a>
```

Пример для урока id=25, вложений id=25 и id=26:
```markdown
📄 <a href="/lessons/25/file/25/">задание.odt</a>
📝 <a href="/lessons/25/file/26/">пример-текста.odt</a>
```

> **ID вложений** можно посмотреть в Django Admin, открыв урок — каждое вложение имеет свой ID.

### Полезные возможности Slidev

```markdown
<!-- Поэтапное появление элементов -->
<v-clicks>

- Появится первым
- Появится вторым
- Появится третьим

</v-clicks>

<!-- Два столбца -->
---
layout: two-cols
---

# Левый столбец
Текст слева

::right::

# Правый столбец
Текст справа

<!-- Формулы LaTeX -->
$$
E = mc^2
$$

<!-- Таблицы -->
| Колонка 1 | Колонка 2 |
|-----------|-----------|
| Значение  | Значение  |
```

## Шаг 3: Собрать презентацию

### Требования
- Node.js (установлен)
- Slidev: `npm install -D @slidev/cli @slidev/theme-default` (уже установлен в проекте)

### Предпросмотр (локально)

```bash
npx slidev slides.md
```

Откроется на http://localhost:3030 — можно редактировать и видеть изменения в реальном времени.

### Сборка

**Важно:** на Windows в Git Bash нужен `MSYS_NO_PATHCONV=1`, иначе пути будут испорчены.

```bash
# Определить переменные
LESSON_DIR="media/lessons/Название_урока"
BASE_URL="/media/lessons/Название_урока/presentation/"

# Собрать
MSYS_NO_PATHCONV=1 npx slidev build slides.md \
  --base "$BASE_URL" \
  --out "$LESSON_DIR/presentation"
```

Конкретный пример:
```bash
MSYS_NO_PATHCONV=1 npx slidev build slides.md \
  --base "/media/lessons/Тест_-_LibreOffice_Writer/presentation/" \
  --out "media/lessons/Тест_-_LibreOffice_Writer/presentation"
```

## Шаг 4: Привязать презентацию к уроку

1. Django Admin → Уроки → открыть урок
2. Раскрыть секцию **Презентация Slidev**
3. Заполнить:
   - **URL презентации:** `/media/lessons/Название_урока/presentation/`
   - **Название презентации:** отображается на кнопке (например, «LibreOffice Writer»)
   - **PDF презентации** (опционально): загрузить PDF-версию для скачивания
4. Сохранить

## Шаг 5: Деплой на сервер

```bash
# На локальной машине: пересобрать если нужно
MSYS_NO_PATHCONV=1 npx slidev build slides.md \
  --base "/media/lessons/Название_урока/presentation/" \
  --out "media/lessons/Название_урока/presentation"

# Закоммитить и запушить (или скопировать вручную)
git add media/lessons/Название_урока/presentation/
git push

# На сервере
ssh admin@192.168.1.199 -p 2222
cd /home/admin/site
git pull
```

Nginx уже настроен раздавать `/media/` — дополнительная настройка не требуется.

## Обновление презентации

1. Отредактировать `slides.md`
2. Пересобрать (Шаг 3)
3. Скопировать на сервер (Шаг 5)

Студенты увидят обновления при следующем открытии — Vite добавляет хеши в имена файлов, поэтому кэш браузера не мешает.

## Частые проблемы

| Проблема | Причина | Решение |
|----------|---------|---------|
| Белый экран | Пути с `/Program Files/Git/` | Использовать `MSYS_NO_PATHCONV=1` |
| 404 при открытии | Файлы не собраны или неправильный URL | Проверить `--base` и `--out` |
| Файлы не скачиваются | Неправильные ID в ссылках | Проверить ID вложений в админке |
| Презентация не обновляется | Старый билд на сервере | Пересобрать и скопировать |
